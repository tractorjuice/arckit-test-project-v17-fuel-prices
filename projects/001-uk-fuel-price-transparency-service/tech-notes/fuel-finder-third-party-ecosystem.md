## Summary

The Fuel Finder open data scheme has rapidly generated a third-party consumer ecosystem. As of 31 March 2026, the DESNZ factsheet confirms 7 named services consuming and displaying Fuel Finder data to UK drivers, with additional API intermediaries providing developer-friendly access. This ecosystem is relevant to the CMA service because the CMA enhanced API (FR-004) must provide distinct value beyond what these third parties already offer.

### CMA Enhanced API Differentiation

The CMA enhanced API must provide capabilities not available from VE3 or third parties:

- **Geospatial search** (lat/lng + radius) -- not available from VE3 API
- **Historical price trends** -- not available from any current source
- **Data quality scoring** (freshness, anomaly flags) -- unique CMA value
- **Bulk GeoJSON download** -- spatial format not offered elsewhere
- **In-car optimised responses** (format=auto) -- purpose-built for Android Auto/CarPlay
- **Unauthenticated read access** -- VE3 API requires OAuth
