## Summary

OAuth 2.0 Client Credentials is the authentication pattern used by the Fuel Finder statutory API for machine-to-machine access. This flow is also the recommended pattern for the CMA service's own API submission endpoint, ensuring architectural consistency between the Fuel Finder platform and the CMA's consumer/producer APIs.

## Key Findings

- **Fuel Finder API Implementation**: POST to `/api/v1/oauth/generate_access_token` with Content-Type `application/x-www-form-urlencoded`, parameters: `grant_type=client_credentials`, `client_id`, `client_secret`, `scope=fuelfinder.read`
- **Token Response**: `access_token` (JWT), `token_type` (Bearer), `expires_in` (3600 seconds)
- **Token Usage**: Include `Authorization: Bearer {token}` header on all API requests
- **Token Refresh**: Automatic refresh on 401/403 with token revalidation; recommended to refresh 60 seconds before expiry
- **Security Practices** (from Fuel Finder developer guidelines and official PDFs): Never expose secrets in client-side code or public repositories; store in environment variables or secure secrets manager (not in source control); use separate credentials per environment and per application; rotate regularly and immediately if compromise suspected; request minimum required scopes (principle of least privilege); keep access tokens short-lived — rely on issuing new tokens; do not log secrets or tokens — mask sensitive values in logs and error messages; HTTPS only (TLS 1.2+) for all calls to token and API endpoints; monitor token usage for anomalies (unexpected IPs, spikes in requests) and revoke if required; protect server-to-server flows — run OAuth on backend services only, never expose secrets to browsers or mobile apps
- **Error Handling**: 401 Unauthorized for missing, expired, or invalid tokens
- **Environments**: Separate test and production credentials (test credentials invalid in production and vice versa)
- **GOV.UK One Login**: Uses OIDC (built on OAuth 2.0) for user authentication; client credentials for service-to-service

## Relevance to Projects

- **Project 001 (UK Fuel Price Transparency Service)**: OAuth 2.0 client credentials is used for: (1) consuming the Fuel Finder statutory API, (2) authenticating large retailer API submissions to the CMA service, (3) service-to-service communication between microservices.
